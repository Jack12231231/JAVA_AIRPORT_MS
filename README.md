# Airport Fleet Manager

## Описание
Проект представляет собой систему управления парком авиационной техники. Основная цель работы — практическое применение принципов объектно-ориентированного программирования (ООП) для создания гибкой и расширяемой архитектуры приложения.

## Ключевые концепции ООП
В данной работе реализованы:
* **Наследование:** Создана иерархия от абстрактного базового класса `Aircraft` к конкретным реализациям: `Plane`, `Helicopter` и `Drone`.
* **Полиморфизм:** Обработка различных типов летательных аппаратов происходит через единый интерфейс базового класса, что позволяет динамически вызывать переопределенные методы (например, `introduce()`).
* **Абстракция:** Базовый класс `Aircraft` объявлен абстрактным, чтобы исключить создание неопределенной техники и задать строгий контракт для наследников.
* **Инкапсуляция:** Состояние объектов защищено модификаторами доступа `private`, взаимодействие происходит через публичные методы доступа (геттеры и сеттеры).

## Функционал
* **Управление данными:** Добавление, редактирование параметров (дальность, нагрузка), удаление и просмотр списка всей техники.
* **Сортировка:** Реализована возможность упорядочивания парка по модели и по дальности полета.
* **Авторизация:** Простая система входа с разделением прав (Администратор/Пользователь).
* **Хранение данных:** Реализовано сохранение и загрузка всей коллекции объектов через механизм **сериализации** в файл `fleet.dat`.
* **Логирование:** Использование **Apache Log4j** для фиксации всех ключевых действий пользователя и отслеживания ошибок ввода.

## Структура проекта
* `Aircraft.java` — Базовый абстрактный класс и классы-наследники.
* `AirportManager.java` — Класс для управления коллекцией (ArrayList) и реализации бизнес-логики.
* `AuthService.java` / `User.java` — Модуль аутентификации.
* `Main.java` — Точка входа, консольное меню и обработка исключений.

## Требования и запуск
1. **JDK 8+**
2. Библиотека **Log4j 1.2.17**.
3. Настроенный файл `log4j.properties` в корне проекта.

### Инструкция:
1. Скомпилируйте файлы: `javac -cp .;log4j-1.2.17.jar com/example/airport/*.java`
2. Запустите приложение: `java -cp .;log4j-1.2.17.jar com.example.airport.Main`

**Данные для входа:**
* **Admin:** `admin` / `admin`
* **User:** можно зарегистрировать через меню.

## Обработка ошибок
В программе реализована защита от «дурака»:
- Проверка корректности числового ввода (перехват `NumberFormatException`).
- Проверка границ индексов при удалении или редактировании.
- Логирование всех исключительных ситуаций для последующего анализа.
